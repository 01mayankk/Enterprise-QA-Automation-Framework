# Documentation Directory

This directory houses detailed markdown guides explaining the architecture, design choices, and execution strategies.

## Contents
* `Architecture.md`: Overview of components and data flow.
* `ArchitectureDecisionRecord.md` (ADR): Justifications for library/tool choices.
* `ExecutionFlow.md`: Detailed test startup and teardown process flow.
* `TestingStrategy.md`: Test design principles and scenario breakdown.
* `ErrorHandling.md`: Logging and exception capturing mechanics.
* `DeploymentGuide.md`: Docker and execution setup instructions.
* `ReportingGuide.md`: HTML generation and screenshot embedding.
* `LEARNING_GUIDE.md`: Recruiter-friendly guide for students to master the project.
* `InterviewQuestions.md`: Expected QA interview questions based on this framework.
* `ResumeContent.md`: High-impact bullets and summaries for resumes.

## Usage
Refer to these markdown documents to understand, run, or present this framework during interviews.
